Other Award Import

Files that are imported must have a .csv extension as such make sure that all your values are separated by commas.
It is important to note that when you specify an award name that already exists in BSA Database this will update the existing data. To keep existing requirements you must specify them in the import.
When adding requirements please list all requirements for an award after the award. To signify a new award leave an empty space between the last requirement and the name of the next award.
The following table will explain what data is valid and how to build our import file. The type and max values are relate to the type of information that will be stored in the associated field (such as: Text or Number) and the maximum number of character allow in each field respectfully.

Award Name (Required)
(type, max)		 The name of the award.
(Text, 254)
Award Image Path (Optional)
(type, max)		 The url of the award image. The path should start at the c:/ directory.
(Text, 254)
Requirement Name (Optional)
(type, max)		 The name of the requirement.
(Text, 44)
Requirement Description (Optional if requirement name is not present)
(type, max)		 The requirement description.
(Text, N/A)

Example

"Award Name", "Award Image Path"
"Requirement Name", "Requirement Description"
"Super Eagle", "c:/user/johnDoe/eagleImage.png"
"1", "work hard"
"2", "be awesome"
""
"Super Star", "c:/user/johnDoe/eagleImage.png"
