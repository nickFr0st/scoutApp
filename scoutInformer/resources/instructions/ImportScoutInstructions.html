Scout Import

Files that are imported must have a .csv extension as such make sure that all your values are separated by commas.
It is important to note that when you specify an scout name that already exists in BSA Database this will update the existing data. To keep existing contacts you must specify them in the import.
When adding contacts please list all contacts for an scout after the scout. To signify a new scout leave an empty space between the last contact and the name of the next scout.
The following table will explain what data is valid and how to build our import file. The type and max values are relate to the type of information that will be stored in the associated field (such as: Text or Number) and the maximum number of character allow in each field respectfully.

Scout Name (Required)
(type, max)		 The name of the scout.
(Text, 90)
Birth Date (Required)
(type, max)		 The scouts birth date. dates must in the form of: MM/DD/YYYY
(Date, N/A)
Current Rank (Required)
(type, max)		 The name of the scouts current advancement rank (star, eagle, etc.). The advancement must already exists in BSA Database
(Text, 256)
Current Rank Date (Required)
(type, max)		 The date that the scout received his current advancement rank. dates must in the form of: MM/DD/YYYY
(Date, N/A)
Contact Type (Optional)
(type, max)		 The type of contact. valid types are:

Home
Mobile
Email

(valid type, N/A)
Contact Name (Required if Contact Type is specified)
(type, max)		 Name of contact (becky, me, etc.).
(Text, 90)
Relation (Required if Contact Type is specified)
(type, max)		 Relation of contact name to the scout (mom, self, etc.).
(Text, 90)
Data (Required if Contact Type is specified)
(type, max)		 contact info. email or phone number (me@coolScouts.com, (123) 456-7890, etc.)
(Text, 255)

Example

"Scout Name","Birth Date","Current Rank","Current Rank Date"
"Contact Type","Contact Name","Relation","Data"
"James","10/15/2001","Star","10/15/2014"
"Mobile","Kim","Mom","(123) 156-4898"
""
